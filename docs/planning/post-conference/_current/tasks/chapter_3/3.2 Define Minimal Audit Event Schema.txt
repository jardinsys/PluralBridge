Chapter 3 Task 3.2
Define Minimal Audit Event Schema

Purpose

This step defines the SQL record shape for minimal Chapter 3 member write audit events.

This is a planning-only step.

It does not add SQL behavior.
It does not add C# runtime behavior.
It does not add browser runtime behavior.
It does not change /api/me behavior.

Audit Table Name

The minimal Chapter 3 member write audit table is:

pb_member_write_audit

This table records successful protected member write operations.
It is not a full audit subsystem.
It is not an audit console.
It is not a duplicate member content store.

Audit Id Field

The audit table includes:

MemberWriteAuditId

MemberWriteAuditId is the primary key.
MemberWriteAuditId is generated by the database or API according to the SQL implementation step.

SystemId Field

The audit table includes:

SystemId

SystemId records the current system resolved from AccessContext.
SystemId is server-owned.
The browser does not supply authoritative SystemId for audit.

AccountId Field

The audit table includes:

AccountId

AccountId records the current account resolved from AccessContext.
AccountId is server-owned.
The browser does not supply audit actor identity.

MembershipId Field

The audit table includes:

MembershipId

MembershipId records the membership used for the protected write when available.
MembershipId may be nullable if the final access context shape requires it.
MembershipId is server-owned.

MemberId Field

The audit table includes:

MemberId

MemberId records the member affected by the write operation.
For member add, MemberId is the created member.
For member edit, MemberId is the updated member.

Operation Field

The audit table includes:

Operation

Operation records the member write operation.

Allowed Chapter 3 operation values are:

member.add
member.edit

The SQL implementation should constrain Operation where practical.

Request Trace Id Field

The audit table includes:

RequestTraceId

RequestTraceId records the request trace id from RequestTraceContext when available.
RequestTraceId is server-owned.
The browser does not supply authoritative audit trace identity.

Created Timestamp Field

The audit table includes:

CreatedAtUtc

CreatedAtUtc records when the audit event was created.
CreatedAtUtc is server-owned.
The SQL implementation should use a server-side default where practical.

Sensitive Content Exclusion

The minimal audit event does not duplicate sensitive member content.

The audit table should not store:

member description body
full member JSON
before-value field dump
after-value field dump
privacy-sensitive member details beyond the target identifiers needed for audit

The audit table identifies the operation and target.
The member table remains the source for member content.

Expected Minimal Column Shape

The minimal schema should include:

MemberWriteAuditId
SystemId
AccountId
MembershipId
MemberId
Operation
RequestTraceId
CreatedAtUtc

Runtime Boundary

This step does not create the audit table.
This step does not add a SQL script.
This step does not add C# audit writer behavior.
This step does not wire audit into member add or member edit.

Done When

Audit table name is chosen.
Audit id field is defined.
SystemId field is defined.
AccountId field is defined.
MembershipId field is defined as nullable if needed.
MemberId field is defined.
Operation field is defined.
Request trace id field is defined.
Created timestamp field is defined.
Sensitive member-content duplication is excluded.
